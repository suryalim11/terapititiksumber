### **1. Tujuan Halaman Detail Pasien**
Halaman ini bertujuan untuk:
- Menampilkan data lengkap pasien.
- Memudahkan pengelolaan janji temu, riwayat medis, dan transaksi.
- Memberikan akses cepat ke fitur penting seperti chat WhatsApp atau email pasien.

---

### **2. Desain Usulan**
#### **Layout Umum**
- **Header**: Nama pasien dan ID pasien.
- **Menu Samping (Opsional)**: Untuk navigasi cepat antara tab-tab utama (misalnya, Profil, Riwayat Medis, Janji Temu, Transaksi).
- **Konten Utama**: Berisi informasi detil pasien, termasuk profil, riwayat medis, janji temu, dan transaksi.
- **Footer**: Tombol aksi penting seperti "Edit Profil" atau "Hapus Pasien".

#### **Komponen Utama**
1. **Profil Pasien**
   - Nama Lengkap
   - ID Pasien
   - Nomor Telepon
   - Email
   - Alamat
   - Jenis Kelamin
   - Tanggal Lahir
   - Umur

2. **Riwayat Medis**
   - Daftar keluhan masing-masing terapi.
   - Tensi saat datang terapi (sebelum dan sesudah).
   - Catatan medis tambahan.

3. **Janji Temu**
   - Jadwal janji temu sebelumnya dan mendatang.
   - Status janji temu (terjadwal, selesai, dibatalkan).

4. **Transaksi**
   - Riwayat pembayaran terkait pasien.
   - Total pendapatan dari pasien.

5. **Aksi Cepat**
   - Tombol untuk membuat janji temu baru.
   - Tombol untuk mengirim pesan melalui WhatsApp atau email.
   - Tombol untuk mengedit atau menghapus pasien.

### **3. Fitur Spesifik**
Berikut adalah fitur spesifik yang dapat diimplementasikan:

#### **a. Profil Pasien**
- Informasi dasar:
  - **Nama Lengkap**
  - **ID Pasien**
  - **Nomor Telepon**
  - **Email**
  - **Alamat**
  - **Jenis Kelamin**
  - **Tanggal Lahir**
  - **Umur** (dapat dihitung otomatis berdasarkan tanggal lahir)
  - **Keluhan** (opsional)

#### **b. Riwayat Medis**
- Data yang perlu ditampilkan:
  - **Daftar Keluhan**: Catatan keluhan pasien setiap kali melakukan terapi.
  - **Tensi Saat Datang Terapi**:
    - Sebelum terapi.
    - Sesudah terapi.
  - **Catatan Medis Tambahan**: Catatan tambahan dari terapis (contoh: hasil pemeriksaan fisik, rekomendasi terapi).

#### **c. Janji Temu**
- Jadwal janji temu:
  - **Tanggal dan Waktu Janji Temu**.
  - **Status Janji Temu** (terjadwal, selesai, dibatalkan).
  - **Deskripsi Janji Temu** (contoh: sesi terapi titik sumber, konsultasi awal).

#### **d. Transaksi**
- Riwayat pembayaran:
  - **Tanggal Transaksi**.
  - **Jumlah Pembayaran**.
  - **Metode Pembayaran** (contoh: Cash, QRIS, Transfer Bank).
  - **Status Pembayaran** (lunas, belum lunas).

#### **e. Aksi Cepat**
- Tombol-tombol penting:
  - **Janji Temu Baru**: Membuat janji temu baru untuk pasien.
  - **Chat WhatsApp**: Mengirim pesan langsung ke pasien melalui WhatsApp.
  - **Email**: Mengirim email kepada pasien.
  - **Edit Profil**: Mengedit informasi pasien.
  - **Hapus Pasien**: Menghapus pasien dari sistem (dengan konfirmasi).

### **4. Implementasi Teknis**
Berikut adalah langkah-langkah implementasi teknis untuk setiap komponen:

#### **a. Profil Pasien**
- **Frontend**:
  - Buat form untuk menampilkan data pasien.
  - Gunakan komponen UI seperti tabel atau kartu untuk tampilan yang rapi.
  - Hitung umur secara otomatis berdasarkan `Tanggal Lahir` menggunakan JavaScript.
- **Backend**:
  - Buat endpoint API untuk mengambil data pasien berdasarkan ID, misalnya `GET /api/patients/:id`.
  - Simpan data pasien dalam database dengan struktur tabel seperti:
    ```sql
    CREATE TABLE patients (
        patient_id VARCHAR(20) PRIMARY KEY,
        name VARCHAR(100),
        phone VARCHAR(20),
        email VARCHAR(100),
        address TEXT,
        gender ENUM('Laki-laki', 'Perempuan'),
        birth_date DATE,
        complaints TEXT
    );
    ```

#### **b. Riwayat Medis**
- **Frontend**:
  - Tampilkan daftar keluhan dan tensi dalam bentuk tabel.
  - Gunakan komponen timeline jika ingin menunjukkan urutan kronologis catatan medis.
- **Backend**:
  - Buat tabel riwayat medis yang terhubung dengan pasien:
    ```sql
    CREATE TABLE medical_history (
        id INT AUTO_INCREMENT PRIMARY KEY,
        patient_id VARCHAR(20),
        complaint TEXT,
        before_treatment_blood_pressure VARCHAR(20),
        after_treatment_blood_pressure VARCHAR(20),
        notes TEXT,
        FOREIGN KEY (patient_id) REFERENCES patients(patient_id)
    );
    ```
  - Endpoint API untuk mengambil riwayat medis: `GET /api/patients/:id/medical-history`.

#### **c. Janji Temu**
- **Frontend**:
  - Tampilkan jadwal janji temu dalam bentuk tabel atau kalender.
  - Gunakan warna atau ikon untuk menandai status janji temu (terjadwal, selesai, dibatalkan).
- **Backend**:
  - Buat tabel janji temu yang terhubung dengan pasien:
    ```sql
    CREATE TABLE appointments (
        id INT AUTO_INCREMENT PRIMARY KEY,
        patient_id VARCHAR(20),
        appointment_date DATETIME,
        status ENUM('Terjadwal', 'Selesai', 'Dibatalkan'),
        description TEXT,
        FOREIGN KEY (patient_id) REFERENCES patients(patient_id)
    );
    ```
  - Endpoint API untuk mengambil janji temu: `GET /api/patients/:id/appointments`.

#### **d. Transaksi**
- **Frontend**:
  - Tampilkan riwayat pembayaran dalam bentuk tabel.
  - Tampilkan total pendapatan dari pasien.
- **Backend**:
  - Buat tabel transaksi yang terhubung dengan pasien:
    ```sql
    CREATE TABLE transactions (
        id INT AUTO_INCREMENT PRIMARY KEY,
        patient_id VARCHAR(20),
        transaction_date DATETIME,
        amount DECIMAL(10, 2),
        payment_method ENUM('Cash', 'QRIS', 'Transfer Bank'),
        status ENUM('Lunas', 'Belum Lunas'),
        FOREIGN KEY (patient_id) REFERENCES patients(patient_id)
    );
    ```
  - Endpoint API untuk mengambil transaksi: `GET /api/patients/:id/transactions`.

#### **e. Aksi Cepat**
- **Frontend**:
  - Tambahkan tombol-tombol aksi cepat di bagian bawah halaman.
  - Integrasi dengan layanan eksternal seperti WhatsApp Business API atau email server.
- **Backend**:
  - Buat endpoint untuk membuat janji temu baru, mengedit profil pasien, atau menghapus pasien:
    ```plaintext
    POST /api/patients/:id/appointments
    PUT /api/patients/:id
    DELETE /api/patients/:id
    ```
  - Pastikan ada konfirmasi sebelum menghapus pasien.

Untuk membangun halaman detail pasien, Anda perlu fokus pada beberapa aspek utama:
