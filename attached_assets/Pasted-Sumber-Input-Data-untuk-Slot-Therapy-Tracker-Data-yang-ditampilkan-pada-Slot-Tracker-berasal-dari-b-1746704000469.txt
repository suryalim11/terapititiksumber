Sumber Input Data untuk Slot Therapy Tracker
Data yang ditampilkan pada Slot Tracker berasal dari beberapa sumber input berikut:

1. Admin Panel / Halaman Therapy Slots
Administrator memasukkan data slot terapi baru melalui form pada halaman Therapy Slots, meliputi:

Tanggal terapi
Waktu slot (misal: 10:00-12:00)
Kuota maksimal pasien
Status aktif/nonaktif
2. Pendaftaran Pasien
Ketika pasien mendaftar melalui link pendaftaran, mereka memilih slot terapi yang tersedia. Setiap pendaftaran akan:

Meningkatkan jumlah currentCount di slot terapi yang dipilih
Membuat record appointment baru dengan status "Scheduled"
Menghubungkan pasien dengan slot terapi spesifik
3. Appointments & Konfirmasi Kehadiran
Admin dapat mengkonfirmasi kehadiran pasien atau mengubah status appointment di halaman Appointments, yang akan:

Mempengaruhi status appointment (Scheduled, Active, Completed, Cancelled)
Appointment yang berstatus Active atau Scheduled dihitung dalam currentCount
Appointment yang Cancelled tidak masuk dalam perhitungan kuota
4. Auto-sync dari Database
Sistem melakukan sinkronisasi otomatis untuk mencocokkan:

Jumlah appointment aktif di database
Nilai currentCount pada slot terapi
Data ini disinkronkan saat admin melakukan refresh atau auto-refresh
Alur Data Lengkap:
Admin membuat slot terapi baru (kuota awal 0 dari total X)
Pasien mendaftar dan memilih slot (kuota bertambah 1)
Jika pasien membatalkan, admin mengubah status appointment (kuota berkurang 1)
Dashboard menampilkan data yang telah diproses dan dihitung persentasenya
API yang Digunakan:
/api/therapy-slots - mengambil data slot terapi
/api/appointments/date/:date - mengambil appointment berdasarkan tanggal
/api/dashboard/stats - mengambil statistik dashboard
/api/dashboard/active-packages - mengambil data paket aktif pasien
Keseluruhan sistem dirancang untuk mempertahankan konsistensi data antara jumlah appointment yang sebenarnya dengan nilai currentCount yang disimpan dalam slot terapi.
