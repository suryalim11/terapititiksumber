---

### **Judul Proyek:**  
Perpanjang Sesi Login Admin untuk Hindari Logout Otomatis  

---

### **1. Penyebab Umum Logout Otomatis**  
- **Session Timeout:** Sesi kadaluarsa terlalu cepat (default biasanya 30 menit).  
- **Token Expired:** Jika menggunakan JWT, token mungkin memiliki masa berlaku pendek.  
- **Cookie Tidak Persisten:** Cookie sesi tidak disimpan di browser setelah tab ditutup.  
- **Server Restart:** Session storage (seperti Redis) terhapus saat server down.  

---

### **2. Solusi Teknis**  

#### **a. Perpanjang Session Timeout**  
- **Di Express.js (Node.js):**  
  ```javascript
  // File app.js atau server.js
  app.use(session({
    secret: 'your-secret-key',
    resave: false,
    saveUninitialized: true,
    cookie: {
      maxAge: 7 * 24 * 60 * 60 * 1000, // 7 hari (dalam milidetik)
      secure: false, // Set true jika pakai HTTPS
      httpOnly: true,
      sameSite: 'lax'
    }
  }));
  ```

- **Di Django (Python):**  
  ```python
  # settings.py
  SESSION_COOKIE_AGE = 604800  # 7 hari (dalam detik)
  SESSION_SAVE_EVERY_REQUEST = True  # Perbarui sesi setiap request
  ```

#### **b. Gunakan "Remember Me" dengan Token**  
- Tambahkan checkbox **"Ingat Saya"** di halaman login:  
  ```html
  <input type="checkbox" name="remember_me"> Ingat Saya
  ```

- **Simpan Token di Database & Cookie:**  
  ```javascript
  // Contoh untuk Node.js
  if (req.body.remember_me) {
    const token = generateSecureToken(); // Buat token unik
    res.cookie('remember_token', token, { maxAge: 7 * 24 * 60 * 60 * 1000 });
    // Simpan token ke database untuk validasi
  }
  ```

#### **c. Perbarui Session Storage**  
- **Gunakan Database untuk Session:**  
  - Install `connect-pg-simple` (Node.js) untuk menyimpan sesi di PostgreSQL:  
    ```bash
    npm install connect-pg-simple
    ```

  - Konfigurasi:  
    ```javascript
    const pgSession = require('connect-pg-simple')(session);
    app.use(session({
      store: new pgSession({
        conString: 'postgres://user:pass@localhost:5432/dbname'
      }),
      secret: 'your-secret',
      cookie: { maxAge: 7 * 24 * 60 * 60 * 1000 }
    }));
    ```

#### **d. Nonaktifkan Cache Browser untuk Halaman Admin**  
- Tambahkan header untuk mencegah caching di sisi client:  
  ```javascript
  app.use((req, res, next) => {
    res.header('Cache-Control', 'no-cache, no-store, must-revalidate');
    res.header('Pragma', 'no-cache');
    res.header('Expires', '0');
    next();
  });
  ```

---

### **3. Contoh Workflow Perpanjang Sesi**  
1. Admin login dengan centang **"Ingat Saya"**.  
2. Server membuat token unik dan menyimpannya di database + cookie.  
3. Setiap request ke halaman admin:  
   - Cek validitas token di cookie.  
   - Jika valid, perbarui waktu kedaluarsa token.  
4. Jika admin menutup browser, cookie tetap ada selama `maxAge` belum tercapai.  

---

### **4. Keamanan Tambahan**  
- **HTTPS:** Pastikan semua komunikasi menggunakan HTTPS untuk melindungi cookie.  
- **Token Rotation:** Perbarui token secara berkala untuk hindari pencurian sesi.  
- **Logout Manual:** Tambahkan tombol logout untuk menghapus cookie/token.  

---

### **5. Testing**  
- Simulasikan login dengan **"Ingat Saya"** → tutup browser → buka lagi → pastikan masih login.  
- Uji durasi sesi dengan tools seperti **Postman** atau **cURL**.  

---

Dengan konfigurasi ini, sesi admin akan bertahan hingga **7 hari** (atau sesuai kebutuhan) tanpa logout otomatis, kecuali pengguna menekan tombol logout atau menghapus cookie. 🔒✨
