Identifikasi Pasien Lama di Link Pendaftaran
Tambahkan field "Nama atau No. WA" di form pendaftaran online.
Jika pasien sudah terdaftar:
Sistem cari data berdasarkan nama atau nomor WA di database.
Jika ditemukan:
Prefill biodata (nama, alamat, tanggal lahir) → pasien hanya perlu pilih sesi.
Tampilkan pesan: "Selamat datang kembali, [Nama]!"
Jika tidak ditemukan:
Pasien lanjut isi form sebagai pasien baru.
Detail Teknis
a. Database Schema
Tambahkan kolom phone di tabel patients untuk menyimpan nomor WA.
Pastikan kolom phone unik untuk menghindari duplikat.

function RegistrationForm() {
  const [query, setQuery] = useState('');
  const [patientData, setPatientData] = useState(null);

  const handleSearch = async () => {
    const response = await fetch(`/api/patients?query=${query}`);
    const data = await response.json();
    setPatientData(data);
  };

  return (
    <div>
      <input
        type="text"
        placeholder="Masukkan Nama atau No. WA"
        value={query}
        onChange={(e) => setQuery(e.target.value)}
      />
      <button onClick={handleSearch}>Cek Data</button>

      {patientData && (
        <div>
          <p>Selamat datang kembali, {patientData.name}!</p>
          <input type="text" defaultValue={patientData.name} readOnly />
          <input type="date" defaultValue={patientData.birthdate} readOnly />
          <textarea defaultValue={patientData.address} readOnly />
        </div>
      )}
    </div>
  );
}
Workflow
Pasien Buka Link Pendaftaran:
Masukkan Nama/No. WA → sistem cek ke database.
Jika ditemukan:
Data biodata otomatis terisi (tidak bisa diedit).
Pasien hanya pilih sesi terapi.
Jika tidak ditemukan:
Isi form lengkap sebagai pasien baru.
Admin Dashboard:
Lihat daftar pasien dengan nomor WA terintegrasi.
Export data pasien ke Excel untuk keperluan follow-up.
Catatan Keamanan
Validasi Nomor WA:
Pastikan format nomor WA valid (contoh: +6281234567890).
Gunakan OTP via WhatsApp untuk verifikasi jika diperlukan.
Privasi Data:
Enkripsi kolom phone di database.
Batasi akses admin ke data sensitif.
