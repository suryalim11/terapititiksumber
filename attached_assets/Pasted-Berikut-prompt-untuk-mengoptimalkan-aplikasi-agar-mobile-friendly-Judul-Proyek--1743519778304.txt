Berikut **prompt untuk mengoptimalkan aplikasi agar mobile-friendly**:

---

### **Judul Proyek:**  
Optimisasi Aplikasi Terapi Titik Sumber untuk Perangkat Mobile  

---

### **1. Prinsip Utama Desain Mobile-First**  
- **Responsive Layout:**  
  - Gunakan grid system (misal: Bootstrap atau CSS Flexbox/Grid).  
  - Pastikan semua elemen (tombol, teks, gambar) menyesuaikan ukuran layar.  

- **Touch-Friendly:**  
  - Ukuran tombol minimal **48x48 pixel** untuk mudah di-tap.  
  - Jarak antar elemen cukup agar tidak terjadi "misclick".  

---

### **2. Perubahan Spesifik untuk Mobile**  
#### **a. Navbar/Header**  
- Ganti menu horizontal dengan **hamburger menu** (☰) untuk layar kecil.  
- Contoh kode:  
  ```html
  <nav class="navbar">
    <button class="hamburger">☰</button>
    <div class="logo">Terapi Titik Sumber</div>
  </nav>
  ```

#### **b. Formulir Pendaftaran**  
- **Input Field:**  
  - Gunakan `inputmode="numeric"` untuk nomor telepon.  
  - Tambahkan `autocomplete="name"` untuk mempercepat pengisian.  
- **Contoh:**  
  ```html
  <input type="text" placeholder="Nama Lengkap" autocomplete="name">
  <input type="tel" placeholder="No. WA" inputmode="numeric">
  ```

#### **c. Tabel/Data**  
- Ganti tabel kompleks dengan **card layout** di mobile.  
  ```html
  <div class="card">
    <div class="card-header">John Doe</div>
    <div class="card-body">
      <p>No. WA: 081234567890</p>
      <p>Sesi: 10:00-11:00</p>
    </div>
  </div>
  ```

---

### **3. Optimisasi Performa**  
- **Kompresi Gambar:**  
  - Gunakan format WebP dan tools seperti **Squoosh** untuk mengurangi ukuran gambar.  
- **Lazy Loading:**  
  - Tambahkan `loading="lazy"` pada gambar yang tidak langsung terlihat.  
- **Minify CSS/JS:**  
  - Gunakan tools seperti **Webpack** atau **PurgeCSS** untuk menghapus kode tidak terpakai.  

---

### **4. Testing & Debugging**  
- **Tools Rekomendasi:**  
  - **Chrome DevTools** (toggle device toolbar).  
  - **Responsively** (aplikasi untuk simulasi multi-device).  
- **Skenario Testing:**  
  - Cek tampilan di layar < 600px (iPhone SE, Samsung Galaxy S20).  
  - Uji kecepatan di jaringan 3G/4G.  

---

### **5. Contoh Kode CSS Responsive**  
```css
/* Mobile-first styling */
.container {
  padding: 20px;
}

/* Tampilan desktop */
@media (min-width: 768px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
}

/* Hamburger menu */
.hamburger {
  display: block;
  font-size: 24px;
}

/* Sembunyikan menu di desktop */
@media (min-width: 768px) {
  .hamburger {
    display: none;
  }
}
```

---

### **6. Fitur Khusus Mobile**  
- **Click-to-Call:**  
  Tambahkan link langsung ke WhatsApp:  
  ```html
  <a href="https://wa.me/6281234567890">Hubungi via WA</a>
  ```

- **PWA (Progressive Web App):**  
  - Izinkan pengguna menginstal aplikasi ke homescreen.  
  - Contoh `manifest.json`:  
    ```json
    {
      "short_name": "Terapi Titik Sumber",
      "start_url": "/",
      "display": "standalone",
      "background_color": "#ffffff"
    }
    ```

---

### **7. Checklist UI/UX Mobile**  
- [ ] Semua teks bisa dibaca tanpa zoom (minimal 16px).  
- [ ] Tidak ada horizontal scrolling.  
- [ ] Waktu loading < 3 detik di jaringan 3G.  
- [ ] Formulir bisa diisi dengan mudah di layar kecil.  

---

Dengan perubahan ini, aplikasi akan lebih nyaman digunakan di smartphone tanpa mengorbankan fungsionalitas! 📱✨
